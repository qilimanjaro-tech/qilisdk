Hardened YAML deserialization against arbitrary code execution. `deserialize` and `deserialize_from` are now safe-by-default: they use a data-only loader that rejects code-bearing tags (`!function`/`!lambda`, which run `dill`) and `!type`/`!PydanticModel`/`!defaultdict`/`!!python/...` tags pointing at names outside a strict allow-list, raising `DeserializationError` instead of executing code at parse time. Reconstructing trusted functions/lambdas now requires the explicit, default-off `trust_code=True` opt-in, which must never be used on untrusted (network/file) input. SpeQtrum job results and payloads, which flow through these functions, are therefore no longer a remote-code-execution vector.
