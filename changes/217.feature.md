New CudaSamplingMethods - CPU and STATE_VECTOR_MGPU, to tell the CudaBackend to use CPU or multiple GPUs, respectively. Multiple GPU statevector simulation requires a very specific setup, check the latest NVIDIA docs to see what versions of the various libraries they recommend. 

The performance of post-processing CUDA samples has also been improved. When calculating an observable cost function from samples, previously .get_probabilities() was used which if the results contained a state (as per the output of CUDA dynamics), would be 2^n values, rather than just the number of samples.
