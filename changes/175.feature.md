Restructured the functionals and results system with a decoupled readout architecture.

### Breaking Changes

- **`Sampling` renamed to `DigitalPropagation`**: The functional now takes only a `circuit` argument. Shot count is specified via readout.
- **`TimeEvolution` renamed to `AnalogEvolution`**: The functional now takes `schedule`, `initial_state`, and `store_intermediate_results`. Observables and shot count are specified via readout.
- **`SamplingResult` and `TimeEvolutionResult` removed**: All primitive functionals now return a unified `FunctionalResult`.
- **`backend.execute()` now requires a `readout` parameter**: Readout methods are decoupled from functionals and passed at execution time.

### New Readout System

A new `qilisdk.readout` module provides three readout methods that are passed to `backend.execute()`:

- `SamplingReadout(nshots)`: Sample the circuit a given number of times.
- `ExpectationReadout(observables)`: Compute expectation values of observables.
- `StateTomographyReadout()`: Return the full quantum state.

### Migration Guide

**Before (digital circuit sampling):**

```python
from qilisdk.functionals import Sampling

sampling = Sampling(circuit=circuit, nshots=500)
result = backend.execute(sampling)
print(result.samples)
```

**After:**

```python
from qilisdk.functionals import DigitalPropagation
from qilisdk.readout import SamplingReadout

functional = DigitalPropagation(circuit)
result = backend.execute(functional, readout=[SamplingReadout(nshots=500)])
print(result.samples)
```

**Before (analog time evolution):**

```python
from qilisdk.functionals import TimeEvolution

te = TimeEvolution(schedule=schedule, initial_state=psi0, observables=[Z(0)], nshots=100)
result = backend.execute(te)
print(result.expected_values)
```

**After:**

```python
from qilisdk.functionals import AnalogEvolution
from qilisdk.readout import ExpectationReadout

evolution = AnalogEvolution(schedule=schedule, initial_state=psi0)
result = backend.execute(evolution, readout=[ExpectationReadout(observables=[Z(0)])])
print(result.expected_values)
```

**Before (variational program):**

```python
vp = VariationalProgram(functional=Sampling(ansatz), optimizer=opt, cost_function=cost_fn)
result = backend.execute(vp)
```

**After:**

```python
vp = VariationalProgram(functional=DigitalPropagation(ansatz), optimizer=opt, cost_function=cost_fn)
result = backend.execute(vp, readout=[SamplingReadout(nshots=1000)])
```

### Result Properties

The unified `FunctionalResult` provides:

- `samples` — Shot counts from `SamplingReadout`
- `probabilities` — Measurement probabilities
- `state` — Quantum state from `StateTomographyReadout`
- `expected_values` — Expectation values from `ExpectationReadout`
- `intermediate_samples`, `intermediate_probabilities`, `intermediate_states`, `intermediate_expected_values` — Lists of intermediate results when `store_intermediate_results=True`

### Other Changes

- Added QiliSim C++ backend support for `DigitalPropagation`, `AnalogEvolution`, and `QuantumReservoir`.
- Added CUDA backend support for the new readout system.
- Added QuTiP backend support for the new readout system.
- Updated SpeQtrum cloud submission to use the new functional and readout API.
- Updated all documentation to reflect the new structure.
