Added LP-format (CPLEX flavor) support to `qilisdk.utils.lp_parser`. A `Model` can now be read from or written to LP text and files, round-trippable through the parser and serializer:

* `from_lp(lp_str)` / `from_lp_file(path)` parse LP-format text into a `Model`.
* `to_lp(model)` / `to_lp_file(model, path)` serialize a `Model` back to LP-format text.

Supported features:

* Linear and quadratic objectives (`Minimize` / `Maximize`, including bilinear and squared terms).
* Linear constraints with senses `<`, `<=`, `=`, `>=`, `>` (and the LP synonyms `=<`, `=>`).
* `Bounds` section: two-sided ranges (`lo <= x <= hi`), one-sided bounds (`x >= lo`, `x <= hi`), fixed values (`x = c`), `free` declarations, and `±infinity` keywords.
* `General` (integer) and `Binary` variable declarations.
* Coefficients and bounds in scientific notation (`1.5e2`, `1e-3`, `2E5`).
* Backslash (`\`) line comments.

Example — build a model in code, serialize it to LP, and read it back:

```python
from qilisdk.core.model import Model, ObjectiveSense
from qilisdk.core.variables import LEQ, Domain, Variable
from qilisdk.utils.lp_parser import from_lp, to_lp, to_lp_file

x = Variable("x", Domain.REAL, bounds=(0, 10))
y = Variable("y", Domain.INTEGER, bounds=(0, 5))

model = Model("example")
model.set_objective(2 * x + 3 * y, label="profit", sense=ObjectiveSense.MAXIMIZE)
model.add_constraint(label="capacity", term=LEQ(x + y, 8))

lp_text = to_lp(model)
print(lp_text)
# Maximize
#  profit: 2 x + 3 y
# Subject To
#  capacity: x + y <= 8
# Bounds
#  0 <= x <= 10
#  0 <= y <= 5
# General
#  y
# End

to_lp_file(model, "example.lp")
round_tripped = from_lp(lp_text)
```

Reading an existing LP file is a one-liner:

```python
from qilisdk.utils.lp_parser import from_lp_file

model = from_lp_file("knapsack.lp")
print(model)
```
