Resolved a circular import between `qilisdk.digital` and `qilisdk.utils.trotterization`. `trotter_evolution` and `_commuting_trotter_evolution` are now imported lazily inside the methods that use them (`TrotterizedSchedule` and `QAOA`), so importing `qilisdk.utils.trotterization` (or `from qilisdk.digital import TrotterizedSchedule`) works regardless of import order.
